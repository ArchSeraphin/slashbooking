1. Préambule

La présente politique de confidentialité décrit comment SlashBooking (ci-après « le plugin ») traite les données personnelles. SlashBooking est un plugin WordPress libre, sous licence GPL v2 ou ultérieure, distribué via slashbox.fr/slashbooking.

Important : SlashBooking est un plugin auto-hébergé. Toutes les données (réservations, comptes Google connectés, e-mails clients) sont stockées dans la base de données WordPress de l'installation où il est activé. L'éditeur du plugin ne collecte ni ne reçoit aucune donnée des installations en production. Le présent document décrit le comportement du plugin lui-même ; chaque exploitant du plugin reste responsable du traitement effectué sur sa propre installation au sens du RGPD.

2. Éditeur du plugin

SlashBooking est édité et distribué par Slashbox, dont les coordonnées de contact sont fournies en section Contact. L'éditeur fournit le code source du plugin mais n'opère aucun service hébergé en lien avec celui-ci.

3. Données collectées par le plugin

Lorsque vous installez SlashBooking sur votre WordPress, le plugin enregistre les données suivantes dans votre propre base de données :

3.1 Données saisies par les visiteurs du formulaire de réservation

  • Nom et prénom (ou raison sociale)
  • Adresse e-mail
  • Numéro de téléphone
  • Adresse postale (si demandée par le service)
  • Notes libres ajoutées par le client
  • Service choisi, date et heure du rendez-vous
  • Horodatage du consentement RGPD coché par le visiteur

Ces données sont stockées dans la table wp_sb_bookings de votre WordPress. Elles sont nécessaires à la finalité du plugin : gérer la prise de rendez-vous et permettre le contact post-réservation.

3.2 Données techniques

  • Adresse IP du visiteur — utilisée uniquement pour le rate-limiting du formulaire (5 demandes par minute par IP), conservée sous forme de hash MD5 dans un transient WordPress expirant en 60 secondes.
  • Token de vérification Cloudflare Turnstile (si activé) — transmis à Cloudflare pour vérification puis détruit. Voir la politique de confidentialité Cloudflare.

4. Utilisation des données Google (Limited Use)

Lorsque l'administrateur du site connecte un compte Google au plugin pour activer la synchronisation Google Calendar, SlashBooking demande l'accès aux scopes OAuth suivants :

  • https://www.googleapis.com/auth/calendar.events — lecture et écriture des événements de calendrier, pour créer/modifier/supprimer les rendez-vous dans le calendrier Google connecté.
  • https://www.googleapis.com/auth/calendar.readonly — lecture de la liste des calendriers du compte, pour permettre à l'administrateur de choisir dans quel calendrier les RDV doivent être synchronisés.

Conformité Google API Services User Data Policy (Limited Use) :

L'utilisation et le transfert vers toute autre application des informations reçues depuis les API Google par SlashBooking se conforment à la Google API Services User Data Policy, y compris les Limited Use requirements.

Concrètement, SlashBooking :

  • N'utilise les données Google que pour la fonctionnalité utilisateur visible qu'est la synchronisation de calendrier (créer des événements à la validation d'un RDV, marquer les créneaux occupés à partir d'événements existants).
  • Ne transfère jamais les données Google à des tiers, sauf si l'utilisateur a explicitement initié le transfert, ou si la loi l'exige (procédure judiciaire, sécurité, fraude).
  • N'utilise pas les données Google à des fins de publicité, de profilage, d'agrégation à des fins marketing ou de vente à des courtiers de données.
  • Ne permet pas la lecture humaine des données Google, sauf si l'utilisateur a explicitement donné son accord pour des données spécifiques (par exemple, journal de synchronisation consulté par l'administrateur dans l'admin WP).

Stockage des credentials Google :

  • Le refresh token Google est chiffré avec libsodium (crypto_secretbox) et stocké dans la table wp_sb_google_accounts de votre WordPress.
  • La clé de chiffrement est définie dans wp-config.php via la constante SLASHBOOKING_ENC_KEY, en dehors de la base de données.
  • Les access tokens (durée de vie 1 h) sont également chiffrés et renouvelés automatiquement.

5. Stockage et localisation

Toutes les données collectées par le plugin sont stockées dans la base de données MySQL/MariaDB de votre installation WordPress. Aucune donnée ne quitte votre infrastructure, à l'exception :

  • Des appels aux API Google nécessaires à la synchronisation Google Calendar (chez Google LLC, USA — couvert par le Data Privacy Framework UE-USA).
  • Des appels de vérification anti-bot à Cloudflare Turnstile (si activé).
  • Des e-mails transactionnels envoyés via votre serveur SMTP WordPress.

6. Partage avec des tiers

L'éditeur du plugin ne reçoit ni ne traite aucune donnée des installations en production de SlashBooking. Le plugin n'envoie aucune télémétrie, aucun ping, aucune statistique d'usage à l'éditeur ni à aucun tiers.

Sur votre installation, les seuls tiers contactés par le plugin sont ceux que vous avez explicitement configurés (Google, Cloudflare, votre serveur SMTP).

7. Durée de conservation

Les réservations sont conservées par défaut 1 095 jours (3 ans) à compter de la date du rendez-vous, durée modifiable par l'administrateur dans les paramètres du plugin (entre 30 et 3 650 jours). Une tâche planifiée mensuelle purge automatiquement les réservations dont la date est antérieure à ce seuil.

Les journaux de synchronisation Google sont purgés quotidiennement après 30 jours.

8. Vos droits (RGPD)

Les visiteurs ayant pris rendez-vous via SlashBooking disposent des droits suivants au titre du RGPD :

  • Droit d'accès aux données les concernant — utiliser Outils → Exporter les données personnelles dans l'admin WP (connecteur officiel implémenté par le plugin).
  • Droit à l'effacement — utiliser Outils → Effacer les données personnelles dans l'admin WP (le plugin anonymise les réservations correspondantes, l'historique métier est préservé sans données identifiantes).
  • Droit de rectification — par e-mail à l'éditeur du site.
  • Droit d'opposition et à la limitation — par e-mail à l'éditeur du site.

9. Révoquer l'accès Google

L'administrateur du site peut révoquer l'accès Google à tout moment :

  • Depuis l'admin WP : SlashBooking → Google → Déconnecter. Le refresh token est supprimé localement.
  • Depuis votre compte Google : myaccount.google.com/permissions → repérez « SlashBooking » dans la liste des applications tierces et cliquez sur « Supprimer l'accès ». Cette opération invalide immédiatement le refresh token côté Google.

10. Cookies et trackers

SlashBooking ne dépose aucun cookie sur le navigateur des visiteurs et n'effectue aucun tracking publicitaire. Aucun script tiers (Google Analytics, Facebook Pixel, etc.) n'est chargé par le plugin.

Si l'anti-bot Cloudflare Turnstile est activé par l'administrateur, Cloudflare peut déposer un cookie technique de session lié à la vérification anti-robot. Voir la politique de cookies Cloudflare.
